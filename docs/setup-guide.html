Universal Browser MCP

Guia de Configuracao para IDEs e Clientes MCP

Iniciar o Servidor

Primeiro, inicie o servidor MCP. Ele ficara rodando e esperando conexoes:

# Entre na pasta do servidor
cd mcp-server

# Inicie o servidor
npm start

O servidor vai iniciar em:

  • HTTP/SSE: http://localhost:8080 (para clientes MCP)
  • WebSocket: ws://localhost:3002 (para a extensao do browser)

Endpoints Disponiveis

Endpoint Metodo Descricao
/sse GET Conexao SSE para clientes MCP
/messages POST Envio de mensagens
/health GET Health check do servidor

2 Configurar seu Cliente MCP

C

Claude Code (CLI)

Terminal / Command Line
1
Abra o arquivo de configuracao:
~/.claude/claude_desktop_config.json
2
Adicione a configuracao: 
{
  "mcpServers": {
    "universal-browser": {
      "url": "http://localhost:8080/sse"
    }
  }
}
3
Reinicie o Claude Code
VS

VS Code

Claude Extension / Copilot
1
Abra Settings (Ctrl+,)
2
Busque por "MCP" ou "Claude MCP"
3
Em settings.json, adicione: 
"claude.mcpServers": {
  "universal-browser": {
    "url": "http://localhost:8080/sse"
  }
}
Alternativa
Ou edite diretamente:
~/.vscode/settings.json
Cu

Cursor

AI Code Editor
1
Abra Cursor Settings (Ctrl+Shift+J)
2
Va em Features → MCP Servers
3
Clique em "Add New MCP Server"
4
Configure: 
"universal-browser": {
  "url": "http://localhost:8080/sse"
}
Arquivo de config
~/.cursor/mcp.json
W

Windsurf

Codeium IDE
1
Abra Windsurf Settings
2
Va em Cascade → MCP Servers
3
Clique em "Add Server" ou edite:
~/.codeium/windsurf/mcp_config.json
4
Adicione a configuracao: 
{
  "mcpServers": {
    "universal-browser": {
      "serverUrl": "http://localhost:8080/sse"
    }
  }
}

3 Testar a Conexao

Apos configurar, teste se esta funcionando:

# Verificar se o servidor esta rodando
curl http://localhost:8080/health

No seu IDE/CLI, peca ao Claude para:

"Crie uma sessao de automacao e navegue para google.com"
Importante
Certifique-se de que a extensao do browser esta instalada e conectada ao servidor WebSocket na porta 3002.
